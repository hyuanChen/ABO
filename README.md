# ABO

> 收藏不是“以后再看”。
>
> 很多时候，收藏就是你正在变成谁的证据。

ABO，`Another Brain Odyssey`，是一个把“收藏、关注、关键词、论文、视频、手记”重新拉回自己手里的桌面工作台。

它不是只会搬运内容的爬虫，也不只是一个把 Markdown 扔进 Obsidian 的工具。  
它更像一条长期路径：把你散落在平台上的兴趣、问题、欲望、研究主题和反复回看的线索，一点点整理成一个真正属于自己的“另一个大脑”。

尤其是小红书收藏。  
你收藏的东西，往往不是最热闹的，而是你舍不得丢、迟早要回来看的。它们比点赞更慢，比转发更私人，也更诚实地反映了你是谁。ABO 想做的，就是把这些收藏从“以后再看”的堆积，变成“我如何理解自己、组织自己、继续出发”的起点。

## 它现在是什么

当前仓库是一个 `Tauri + React + TypeScript + FastAPI` 的桌面应用，已经把“内容抓取 -> 入库 -> 回看 -> 继续追踪”这条链路搭起来了。

第一次使用或想快速理解全局入口时，先看 [ABO 使用参考指南](docs/abo-user-guide.md)。它按当前侧边栏、主动工具和配置流程整理了完整上手路线。

侧边栏不是装饰，它本身就是你在 ABO 里的工作路径：

- `角色主页`：看自己的能量、SAN、代号、成长状态
- `今日情报`：汇总所有模块抓回来的内容卡片
- `数据洞察`：看阅读、活跃度、模块表现与趋势
- `情报库`：把内容沉到自己的资料空间里
- `文献库` / `Wiki` / `手记`：继续做整理、关联、沉淀
- `自动化模块`：把小红书、B 站、论文源、知乎、文件夹监控这些入口接进来

换句话说，ABO 不是单点工具，而是一套“先收集，再理解，再形成自己的知识系统”的桌面操作台。

## 为什么从收藏开始

因为收藏最像一个人的内在地图。

你会收藏什么，说明你反复被什么打动。  
你舍不得删掉什么，说明你想靠近什么样的生活、能力和身份。  
你总在相似主题上停下来，说明那里就是你真正的长期问题。

所以在 ABO 里，收藏不是边角功能，而是入口。  
从收藏出发，你可以看见自己的主题；从主题出发，你可以反推值得长期关注的人；从这些人和这些关键词出发，你又能慢慢长出一个更清晰、更稳定、更可回看的第二大脑。

这就是 `Another Brain Odyssey` 里“Odyssey”的意思：  
不是一次抓取，而是一段长期返回自己的旅程。

## 当前已经能做什么

### 1. 小红书主工作流已经成型

当前小红书工具页的标题是：

`小红书分析工具`

副标题对应的就是现在最核心的两条主链路：

`关注监控（含关键词扫描）、收藏专辑抓取，一键获取 Cookie；普通小红书内容保存到情报库 xhs，专辑内容保存到情报库 专辑`

也就是说，当前版本不是围绕“单条复制”设计的，而是围绕以下两件事：

- 把收藏专辑系统化抓进本地
- 把关注流和关键词扫描变成持续追踪

### 2. 侧边栏里的自动化模块已经接起来了

当前前后端已经接入或预留了这些模块能力：

- `小红书`

- `Bilibili`

- `ArXiv`

- `Semantic Scholar`

  你可以把它理解成：小红书负责生活流、兴趣流和身份线索，B 站负责视频知识与收藏整理，论文模块负责学术追踪，文件夹监控负责把你本地已有材料也纳入系统。它们最后都会回到同一个 ABO 工作台里。

## 小红书能力说明

这是当前仓库里最完整、也最有辨识度的一条链路。

### 当前 UI 里可直接使用的两大标签

#### `收藏专辑抓取`

这一页现在已经支持：

- 一键读取浏览器 Cookie，并保存到 ABO 全局配置
- 后台读取小红书收藏专辑列表
- 专辑多选
- `增量` / `全量` 两种抓取模式
- 可选下载：
  - 图片
  - Live 图动态片段
  - 视频
  - 评论（实验中）
- 指定“最近 N 天”做过滤
- 低频分批恢复抓取
- 使用当前 Edge 独立窗口执行，降低干扰
- 展示专辑读取进度、当前专辑进度、过滤明细、失败情况
- 关注监控 / 搜索结果保存到你的情报库 `xhs/` 目录
- 收藏专辑抓取保存到你的情报库 `专辑/` 目录

这个流程适合做的事很明确：  
先把你过去的收藏系统化导入本地，拿到一个真正可检索、可回看、可继续分析的个人样本库。

#### `关注监控`

这一页现在已经支持：

- 关注流扫描
- 关键词扫描
- 按点赞数 / 发布时间排序
- 最低点赞阈值
- 搜索上限控制
- 把搜索结果直接保存到情报库 `xhs/`
- `收藏反推博主`
- 把候选博主同步到关注推送池
- 关键词推送配置
- 关注流补充开关
- 最近任务历史与状态回看

这条链路更像“动态更新层”。  
收藏专辑告诉你“你过去反复留下了什么”，关注监控告诉你“这些主题现在又长到哪里去了”。

### 当前后端和脚本层已经可用，但主 UI 还没作为主入口展示的能力

虽然当前界面把重点收敛在“收藏专辑抓取”和“关注监控”两条主工作流上，但底层其实已经有这些能力：

- 单条笔记入库
- 批量链接入库
- 评论抓取
- 搜索结果预览直接保存

这些能力目前主要在后端接口和 `scripts/xhs/` 里可直接使用，后续也很容易重新挂回 UI。

## 小红书抓取链路

ABO 当前的小红书设计不是“纯请求式爬虫”，而是明显偏向真实浏览器优先：

1. 优先走本地浏览器扩展 bridge
2. bridge 不可用时，回退到浏览器 CDP
3. 部分详情链路再回退到后端 HTML 解析兜底

当前仓库里的 `XHS Bridge` 扩展就在：

```text
extension/
```

它的目标不是绕开真实浏览器，而是尽量复用真实登录态、真实页面状态和更接近人工操作的路径。

这也是为什么 ABO 的小红书抓取更适合做“长期、低频、可恢复”的知识库导入，而不是暴力扫站。

## 保存结果会去哪里

你配置好情报库路径之后，ABO 会把小红书内容沉到：

```text
<你的 Vault>/xhs/
```

同时，应用还会把一些运行信息写到本地：

```text
~/.abo-config.json
~/.abo/
```

其中包括：

- 全局配置
- Cookie 相关配置
- 小红书任务历史
- 模块运行状态

这意味着你不是只得到一次性结果，而是得到一个可续跑、可回看、可继续扩展的本地系统。

## 快速开始

### 1. 环境准备

建议本机具备：

- `Python 3.11+`
- `Node.js 20+`
- `Rust` 与 `Tauri` 开发环境
- 已登录小红书的 `Edge` 或 `Chrome`
- 一个你愿意长期沉淀内容的本地目录，最好是 `Obsidian Vault`

### 2. 安装依赖

```bash
npm install
python3 -m pip install -r requirements.txt
```

### 3. 启动桌面应用

最省事的方式是直接跑 Tauri 开发环境：

```bash
npm run tauri:fresh-dev
```

这个脚本会清掉旧的前后端端口，然后启动桌面应用。  
Tauri 会通过 `beforeDevCommand` 自动拉起：

- Python 后端：`http://127.0.0.1:8765`
- Vite 前端：`http://localhost:1420`

如果你只想单独调试前后端，也可以分别运行：

```bash
python3 -m abo.main
npm run dev
```

### 4. 第一次使用建议这样走

1. 启动应用，先配置你的 Vault 路径
2. 从侧边栏进入自动化模块里的 `小红书分析工具`
3. 点击右上角 `配置 Cookie`
4. 优先使用 `一键获取浏览器 Cookie`
5. 先跑一次 `收藏专辑抓取`，把过去的收藏导进来
6. 再去 `关注监控` 打开关键词扫描、关注流扫描和推送配置
7. 最后回到 `情报库`、`手记`、`Wiki` 继续整理这些内容

如果你要跑更稳定的小红书浏览器链路，仓库里已经提供了辅助脚本：

```bash
bash scripts/xhs/open_browser_with_extension.sh
```

它会：

- 启动 Edge 或 Chrome
- 自动加载 `extension/`
- 打开 CDP 端口
- 使用单独的浏览器 profile

首次使用时，直接在这个浏览器实例里登录小红书即可。

## 一条推荐使用路径

如果你是第一次认真整理自己的收藏，建议不要一上来就追求“抓全世界”，而是先走下面这条路径：

1. 先抓收藏专辑  
   把你已经默默留了很久的东西收回来。

2. 再做收藏反推博主  
   看看你真正反复靠近的是哪些人、哪些表达风格、哪些主题。

3. 再开关注监控和关键词扫描  
   把“过去的自我线索”延伸成“现在仍在发生的更新流”。

4. 最后回到情报库和手记  
   不只是存下来，而是写下：为什么这条对你重要，它和你正在做的事有什么关系。

这样用，ABO 才不是一个收藏搬运器，而会慢慢长成你的第二大脑。

## 当前边界与注意事项

这一部分很重要，因为它决定了你应该怎样和系统相处。

- 小红书反爬严格，`收藏专辑抓取` 本身就是慢链路，当前 UI 已明确提示大约 `10s 一条`
- 收藏专辑抓取时，小红书窗口不能被完全遮挡，至少要露出一点，否则页面滚动可能失败
- 如果出现限流，优先做这几件事：
  - 等一会儿再试
  - 更新 Cookie
  - 切换 IP
  - 重新登录
  - 使用低频分批模式继续抓取
- 评论、楼中楼、视频等能力更慢、更容易触发风控，适合按需打开，不适合默认暴力全开
- 仓库里的浏览器辅助脚本目前偏 `macOS + Edge/Chrome` 工作流，其他系统可以沿用思路手动启动

如果你把 ABO 当作“长期跑、慢慢长”的系统，而不是“十分钟抓完整个平台”的脚本，它会更稳定，也更接近它本来的设计方向。

## 项目结构

```text
ABO/
├── abo/                # FastAPI 后端、模块系统、抓取逻辑、调度器
├── src/                # React 前端与应用 UI
├── src-tauri/          # Tauri 壳与桌面端配置
├── extension/          # 小红书浏览器 bridge 扩展
├── scripts/xhs/        # 小红书辅助脚本
├── tests/              # Python 测试
└── test-validation/    # 验证脚本与样例数据
```

## 这个项目更适合谁

ABO 更适合下面这类人：

- 有大量收藏，但越来越不想把收藏夹当垃圾堆的人
- 想把小红书、B 站、论文和本地资料统一到一个系统里的人
- 想从“我看过很多”走到“我知道自己为什么反复看这些”的人
- 想做个人知识库，但又不想只做冷冰冰整理的人

因为它关心的不只是“你抓了多少”，而是“这些内容为什么会持续出现在你的人生里”。
