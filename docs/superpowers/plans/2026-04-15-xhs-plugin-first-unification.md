# 小红书插件优先抓取统一计划

日期：2026-04-15

## 目标

把小红书工具里的两条列表抓取链路统一到和“专辑抓取”同一个方向：

1. 优先走本地 `bridge -> 浏览器扩展 -> 真实页面 MAIN world state`
2. 页面类型明确区分，不再混用搜索页和关注流的 state ready 条件
3. 失败时再进入浏览器级兜底链路
4. 前端入口收敛，`关键词扫描` 作为 `关注监控` 的子能力，不再单独占一个一级按钮

## 当前问题

### 现状偏差

- 专辑抓取已经形成了相对稳定的“插件优先”主链路。
- 笔记搜索和关注监控虽然也接了扩展，但实现还是偏轻：
  - 搜索 / 关注流共用同一个列表抓取函数。
  - `wait_for_xhs_state` 原先写死为 `search`，关注流页没有按 `feed` 类型等待。
  - 前端把 `笔记搜索` 单独作为一级入口，和 `关注监控` 的产品心智分裂。
  - 搜索 / 关注流的抓取配置实际上复用了专辑页的扩展端口和独立窗口状态，但 UI 没有明确暴露。

### 风险

- 关注流页 state ready 判定不准确时，容易在页面未稳定时开始滚动，导致空结果或结果不全。
- 搜索和关注流共用一套抽取逻辑但没有显式 page kind，后续很难继续贴近专辑抓取的稳定化改造。
- 前端入口分裂后，用户不知道两条链路其实共用同一套抓取环境，排障成本高。

## 分阶段方案

### Phase 1：统一列表抓取骨架

目标：先让搜索 / 关注流的“插件优先”主链路在结构上和专辑抓取一致。

已完成：

- 抽出统一入口 `_extract_cards_with_plugin_priority(...)`
- 搜索显式使用 `page_kind="search"`
- 关注流显式使用 `page_kind="feed"`
- `wait_for_xhs_state` 按页面类型等待
- 卡片抽取表达式、`xsec_source` 默认值、卡片转 `XHSNote` 的逻辑统一

验收点：

- 搜索任务使用搜索页 state ready
- 关注流任务使用 feed 页 state ready
- 扩展失败时能回退到 Playwright

### Phase 2：把列表抓取进一步贴近专辑链路

目标：把专辑抓取里更稳定的控制手段下沉到搜索 / 关注流。

待做：

- 为搜索 / 关注流补更细粒度的进度状态
- 评估是否引入与专辑一致的前台辅助 / 标签脉冲策略
- 增加二级兜底：Playwright 失败时进入 CDP 或更轻的 state 读取链路
- 把页面风险识别和错误码表达统一成和专辑一样的格式

验收点：

- 风控、登录失效、页面空载入能给出明确错误类型
- 搜索 / 关注流的后台任务日志能看懂具体停在哪一步

### Phase 3：产品入口统一

目标：把“关键词扫描”完全并入“关注监控”。

已完成：

- 顶部一级入口移除 `笔记搜索`
- `关键词扫描` 作为 `关注监控` 子区块展示
- 关注流扫描和关键词扫描共用同一套桥接配置 UI
- 任务显示文案改为 `关键词扫描` / `关注流扫描`

待做：

- 评估是否把搜索结果和关注流结果合并到同一结果面板
- 给两种来源增加更统一的来源标签和保存策略

### Phase 4：和 tracker 模块彻底对齐

目标：让工具页手动抓取和 `xiaohongshu-tracker` 定时抓取共享更多底层配置。

待做：

- 为 tracker 模块补充显式的 bridge / extension / dedicated window 配置项
- 统一 `follow_feed + keyword_search` 的执行诊断输出
- 保证工具页、任务路由、默认模块三处的执行链路和命名一致

验收点：

- 手动抓取和定时任务的链路说明一致
- 定位问题时不需要分别猜测“工具页”和“模块任务”用了不同逻辑

## 本轮已落地改动

### 后端

- `abo/tools/xiaohongshu.py`
  - 新增统一列表抓取入口
  - 搜索 / 关注流显式区分 `page_kind`
  - 扩展卡片抽取表达式参数化
  - 关注流不再误用搜索页的 state ready 条件

- `abo/routes/tools.py`
  - 任务摘要文案改成 `关键词扫描` / `关注流扫描`

### 前端

- `src/modules/xiaohongshu/XiaohongshuTool.tsx`
  - 顶部 tab 去掉 `笔记搜索`
  - `关键词扫描` 并入 `关注监控`
  - 显式展示搜索 / 关注流共用的桥接配置
  - 任务历史名称同步调整

### 测试

- 新增针对列表抓取 page kind 和 fallback 的单测
- 保留 tracker 现有聚合逻辑测试

## 下一轮建议

优先顺序：

1. 给搜索 / 关注流加“专辑风格”的进度上报
2. 把列表抓取失败链路从 Playwright-only 扩到 `Playwright -> CDP`
3. 评估是否让 tracker 也默认进入 dedicated window 模式
