# ABO 使用参考指南

ABO 是一个本地研究工作台。它把论文、小红书、B 站、收藏、关注流、手记和 Wiki 放进同一条流程：

```text
注意力 -> 知识 -> 入库 -> Wiki / 助手推进 -> 手记复盘
```

第一次使用时，不需要一次性理解所有功能。先完成初始化向导，再按本文的路线每天使用即可。

新用户第一次打开 ABO 时会自动进入 UI 向导。向导右上角可以跳过；如果之后想重新学习或关闭下次显示，可以到 `设置 -> 通用 -> 新手向导` 调整。

## 10 秒钟基础配置

1. 选择情报库和文献库。
   情报库保存社媒、收藏、网页、手记等内容；文献库保存论文。第一次建议都指向同一个 Obsidian Vault，熟悉后再拆开。

2. 连接小红书和 B 站 Cookie。
   在向导、设置页或对应主动工具里点击“一键配置”。ABO 会复用当前工具页的浏览器 Cookie 获取逻辑，读取后自动测试并写入全局配置。

3. 配置学术和 AI。
   arXiv 不需要 API Key。Semantic Scholar API Key 可选，留空也能先使用。默认后台 Agent 使用 Codex；Claude 暂不支持。

4. 设置每日情报时间。
   默认是 09:00。论文、小宇宙、知乎按该时间调度；小红书和 B 站会提前 30 分钟预抓取。

5. 先用主动工具试跑。
   定时任务本质上只是“到点执行主动工具已有链路”。第一次应先在小红书、B 站或论文工具页手动跑一次，确认预览、保存路径和过滤配置都正确。

## 核心功能怎么用

ABO 的核心不是“打开一个开关等它自己工作”，而是五个动作连起来：先配置基础环境，再主动爬取试跑，再把可持续来源保存成关注监控，之后把数据入库维护，最后让助手基于这些本地材料继续推进。

### 1. 配置基础环境

入口：`初始化向导` 或 `设置 -> 通用`

你需要先配置这些内容：

1. `情报库路径`
   小红书、B 站、收藏、网页、知乎、手记等非论文内容会保存到这里。建议选一个长期使用的 Obsidian Vault。

2. `文献库路径`
   arXiv、Semantic Scholar 和 Follow Up 论文会保存到这里。第一次可以和情报库共用同一个 Vault，后续再拆。

3. `小红书 Cookie`
   用于收藏专辑抓取、主动爬取、关注流扫描、固定博主监控。优先点击“一键配置”，并确认浏览器已经登录小红书。

4. `B 站 Cookie`
   用于关注流、收藏夹、智能分组和固定 UP 监控。优先点击“一键配置”，并确认浏览器已经登录 B 站。

5. `AI 助手`
   默认后台 Agent 使用 Codex。Claude 暂不支持，这个设置会影响助手入口，也影响后端统一 AI 调用链路。

6. `论文 AI 帮读评分`
   如果开启，新抓取论文会调用后台 AI 生成评分、短摘要、标签和核心创新。如果只想先稳定抓取元数据，可以先关闭。

7. `Semantic Scholar API Key`
   可选。少量 Follow Up 可以留空；大量使用建议填写自己的 Key。

8. `默认推送时间`
   默认是 09:00。论文、小宇宙、知乎按这个时间调度；小红书和 B 站会提前 30 分钟预抓取。

配置完成后的判断标准：

- 侧边栏底部显示 `库已连接`。
- 小红书和 B 站 Cookie 显示已连接或测试通过。
- 今日情报调度开关符合你的预期。
- 模块管理里需要的模块处于启用状态。

### 2. 主动爬取获取数据

入口：侧边栏 `主动工具`

主动爬取是第一次使用时最重要的一步。它的目的不是马上全量抓取，而是先确认“能抓到、能预览、能保存、保存位置正确”。

小红书主动爬取：

1. 打开 `小红书工具`。
2. 点击右上角 `配置 Cookie`，优先使用一键获取。
3. 先用 `收藏专辑抓取` 导入一个小专辑或少量内容。
4. 再用 `主动爬取` 测试关键词、笔记链接或指定博主。
5. 在结果卡片里检查标题、作者、时间、摘要、图片和保存按钮。
6. 先保存少量结果，再去 `情报库` 检查是否进入 `xhs/` 或 `专辑/`。

B 站主动爬取：

1. 打开 `哔哩哔哩工具`。
2. 点击右上角 `配置 Cookie`，优先使用一键获取。
3. 在 `动态追踪` 里跑全关注流、关键词或指定 UP。
4. 在 `收藏整理` 里读取收藏夹并预览内容。
5. 检查动态卡片、视频卡片、作者、链接和摘要是否正确。
6. 先保存少量结果，再去 `情报库` 检查 B 站内容目录。

论文主动爬取：

1. 打开 `论文追踪` 或 `arXiv API`。
2. 如果是找某个方向，用 arXiv 关键词或分类搜索。
3. 如果是追踪某篇论文的后续研究，用 Semantic Scholar 输入论文标题或 arXiv ID。
4. 检查标题、作者、摘要、PDF、图片和 Introduction 是否抓到。
5. 保存少量论文到 `文献库`。
6. 去 `文献库` 确认 Markdown、PDF 和图片路径是否正确。

主动爬取后的判断标准：

- 工具页能正常返回卡片。
- 卡片里有可判断的信息，而不是只有空标题或空摘要。
- 保存后能在情报库或文献库找到文件。
- 如果主动工具跑不通，先不要依赖定时任务。

### 3. 设置关注监控获取持续数据

入口：主动工具里的 `关注监控`，以及 `论文追踪 -> 关注监控`

关注监控用于把一次性搜索变成每天自动更新的数据来源。ABO 的定时任务应该复用主动工具已有链路，所以监控配置也应优先在主动工具里完成。

小红书关注监控：

1. 打开 `小红书工具 -> 关注监控`。
2. 配置 `关键词扫描`：适合追踪主题，例如“机器人学习”“效率工具”“留学申请”。
3. 配置 `关注流扫描`：适合从你已关注的人里筛选相关内容。
4. 配置 `固定博主`：适合持续追踪少数高价值作者。
5. 设置抓取上限、最近天数、排序方式、最低点赞阈值和是否启用关键词过滤。
6. 保存配置后，先手动执行一次，确认 Feed 里能看到新卡片。

B 站关注监控：

1. 打开 `哔哩哔哩工具 -> 关注监控`。
2. 配置 `关注关键词监控`：从关注流中按关键词筛选动态。
3. 配置 `智能分组追踪`：按 ABO 生成或你维护的分组监控 UP。
4. 配置 `固定监督 UP`：直接盯住某些 UP 主的最新动态。
5. 保存后手动运行一次，确认动态卡片可以进入今日情报。
6. 如果 Cookie 失效，先回到工具页重新一键配置。

论文关注监控：

1. 打开 `论文追踪 -> 关注监控`。
2. 配置 `arXiv 关键词监控`：适合长期追踪研究方向。
3. 配置 `Follow Up 监控`：适合追踪某篇源论文之后的新工作。
4. 设置分类、关键词、最大结果数、最近天数和排序方式。
5. 保存后立即运行一次，确认论文能进入今日情报或文献库。

监控保存后的判断标准：

- 模块管理里对应模块处于启用状态。
- 模块详情能看到监控内容、最近记录、上次运行和下次运行。
- 今日情报能按论文、社媒、小红书、B 站、关键词或作者过滤到对应结果。
- 需要改关键词、作者、UP 主或抓取策略时，回到主动工具，不要只在模块管理里改。

### 4. 入库与维护

入口：`今日情报`、`情报库`、`文献库`、`Wiki`

入库维护的目标是把“今天看到的卡片”变成“之后还能找到、能关联、能继续分析”的本地知识库。

今日情报处理顺序：

1. 打开 `今日情报`。
2. 先用过滤树缩小范围：全部情报、论文追踪、社媒关注、小红书、B 站、具体关键词或作者。
3. 对每张卡片做判断：保存、已读、跳过、打开原文，或交给助手分析。
4. 不值得保留的内容及时跳过，避免 Feed 变成新的收藏夹垃圾堆。
5. 值得保留的社媒内容保存到情报库，论文保存到文献库。

情报库维护：

1. 打开 `情报库`。
2. 用泡泡视图看哪些文件夹内容最多。
3. 用列表视图定位具体文件。
4. 需要深度整理时，用 `在 Obsidian 中打开` 或 `在 Finder 中打开`。
5. 如果目录不对，先检查情报库路径和工具页保存配置。

文献库维护：

1. 打开 `文献库`。
2. 确认论文 Markdown、PDF、图片和摘要是否进入正确目录。
3. 用泡泡视图看研究方向堆积情况。
4. 用列表视图找具体论文。
5. 需要改路径时用文献库页面的 `更改路径`。

Wiki 维护：

1. 打开 `Wiki`。
2. 用 `Internet Wiki` 整理情报库里的对象、主题、项目、平台和方法。
3. 用 `Literature Wiki` 整理论文页、主题页、方法线和研究主线。
4. 进入 Wiki 后，用二级侧边栏的搜索、概览和分类树定位页面。
5. 生成或更新 Wiki 后，再让助手基于这些页面做总结、对比或下一步计划。

入库后的判断标准：

- Feed 中的重点卡片不只是“已读”，而是进入了情报库或文献库。
- Wiki 中能看到主题页、对象页或论文页。
- 助手能引用这些本地材料继续工作。
- 你能在 Obsidian 或 Finder 中找到原始 Markdown。

### 5. 助手使用

入口：侧边栏 `助手`

助手不是单纯聊天入口。正确用法是让它基于 ABO 已经抓取、入库和生成的上下文继续推进研究。

开始前先看：

1. `上下文概览`
   确认今日情报、知识库状态、数据洞察是否已经就位。

2. `常用助手`
   选择论文调研、Wiki 维护、情报推进等任务模板，不要每次从空白聊天开始。

3. `最近对话`
   对连续任务直接接着上次上下文继续，避免重复解释背景。

适合交给助手的任务：

1. “把今天小红书和 B 站关于某个主题的卡片合并成 3 条可行动 insight。”
2. “基于这批 Follow Up 论文，整理源论文之后大家解决了哪些 challenge。”
3. “把 Literature Wiki 里的某个主题页更新成更清晰的研究脉络。”
4. “从今日情报里挑出和我当前项目最相关的 5 条，并给出下一步阅读顺序。”
5. “把这几条内容转成明天的待办和实验计划。”

给助手写指令时要包含：

1. 输入来源：今日情报、某个模块、某个 Wiki 页面、某个文献库路径或一段粘贴内容。
2. 判断标准：按相关性、可执行性、新颖性、实验价值或写作价值筛选。
3. 期望产出：总结、对比表、Wiki 草稿、研究 idea、待办、阅读计划或下一步实验。
4. 限制条件：不要泛泛总结、不要重复已有观点、优先引用本地材料。

助手使用后的判断标准：

- 输出能直接变成 Wiki 页面、手记、待办或下一轮研究动作。
- 它引用的是你的本地材料，而不是泛泛常识。
- 对长期任务，你能从最近对话继续推进。

## 侧边栏地图

### 主工作区

`角色主页`
看能量、SAN、今日待办、时间线、六维能力评估、技能树和成就徽章。这里回答“我最近的研究节奏和状态怎么样”。

`助手`
让 Codex 结合今日情报、Wiki、文献库和最近对话继续工作。页面主要分为常用助手、最近对话、对话推进、快捷跳转和上下文概览。

`今日情报`
所有定时和手动抓取的内容卡片先到这里。当前页内主要分组包括情报 Feed、分类与过滤、今日重点和卡片列表。过滤树包含全部情报、论文追踪、社媒关注和其他来源。

`数据洞察`
查看 Today、Intelligence Mirror、Wellness Trends、Engagement Depth、Research Focus 和 30-Day Activity。

`情报库`
查看社媒、收藏和网页等非论文内容。支持泡泡视图、列表视图、在 Obsidian 中打开、在 Finder 中打开和更改路径。

`文献库`
管理论文和文献资料。未配置时会先显示配置路径；配置后支持泡泡视图、列表视图、Obsidian/Finder 打开和更改路径。

`Wiki`
把情报库和文献库生成 Wiki。入口包括 Internet Wiki 和 Literature Wiki，进入后可在页面视图和脑图之间切换。

`手记`
记录每日思考，并沉淀周记总结、月度回顾、年度总结。

### 自动化模块

`模块管理`
只负责运行、暂停、诊断和查看效果，不负责复制一套配置。关键词、关注源、Cookie 和监控细节应回到对应主动工具页调整。

模块管理页主要分组：

- 运行中模块
- 待看内容
- 近 7 天浏览
- 本周新增
- 搜索模块、关键词或订阅源
- 筛选：全部、运行中、暂停、异常
- 偏好学习

模块详情弹窗主要分组：

- 运行概览
- 历史记录
- 监控内容
- 最近记录
- 运行健康
- 快速修复
- 运行诊断

当前侧边栏默认展示的核心模块：

- arXiv 论文追踪
- Semantic Scholar 追踪
- 小红书追踪
- 哔哩哔哩追踪

默认隐藏的 TODO 模块：

- 小宇宙追踪
- 知乎追踪
- 文件夹监控

这些可以在设置里的“隐藏模块开关”重新放出。

### 主动工具

`小红书工具`
主要分组为收藏专辑抓取、主动爬取、关注监控。右上角可以配置 Cookie。适合先导入收藏专辑，再配置关键词、关注流和固定博主监控。

`哔哩哔哩工具`
主要分组为动态追踪、收藏整理、关注监控。右上角可以配置 Cookie。适合按全关注流、智能分组或指定 UP 预览动态，再选择写入情报库。

`论文追踪`
主要分组为后续论文、AI 领域论文、关注监控。适合做 Semantic Scholar Follow Up、arXiv 关键词监控和论文入库。

`arXiv API`
主要分组为搜索条件和搜索结果。适合临时检索、批量浏览和保存论文。

`健康管理`
主要分组为今日状态校准、提醒中心、今天的节律轨迹、习惯与想成为的人、快速读懂今天缺什么、每周复盘、最近两周恢复曲线、推进与打卡的关系。

### 底部与全局入口

`库连接状态`

`设置`
设置页内部侧边栏分为通用、开发调试、关于。

通用主要分组：

- 情报调度
- 监控词条
- 隐藏模块开关
- 今日情报
- AI 助手
- 外观设置
- 炫酷展示模式
- 角色头像
- 键盘快捷键

开发调试主要分组：

- Feed 流测试
- 抓取元数据账本

关于主要分组：

- 关于
- 技术栈

## 今日情报怎么处理

1. 打开 `今日情报`。
2. 先用 `全部情报`、`论文追踪`、`社媒关注` 等过滤树缩小范围。
3. 对卡片做保存、已读、跳过或进一步追问。
4. 社媒内容保存到情报库，论文保存到文献库。
5. 处理完一批后进入 `Wiki` 生成或更新页面。
6. 回到 `助手` 或 `手记`，把今天的线索变成下一步实验、阅读清单或复盘。

## 论文工作流

临时找论文：

1. 打开 `arXiv API`。
2. 设置搜索条件。
3. 浏览结果。
4. 对值得读的论文保存到文献库。

追踪一个研究方向：

1. 打开 `论文追踪`。
2. 在 `AI 领域论文` 中试跑关键词。
3. 到 `关注监控` 保存关键词监控。
4. 第二天从 `今日情报` 处理新论文。

追踪一篇论文的后续工作：

1. 打开 `论文追踪`。
2. 切到 `后续论文`。
3. 输入论文标题或 arXiv ID。
4. 预览结果并保存有价值的 Follow Up。
5. 到 Literature Wiki 生成主题页或论文页。

## 小红书工作流

第一次建议顺序：

1. 打开 `小红书工具`。
2. 点击右上角 `配置 Cookie`，优先使用一键获取。
3. 先跑 `收藏专辑抓取`，把已有收藏导入本地。
4. 再跑 `主动爬取`，测试关键词、链接和保存结果。
5. 最后配置 `关注监控`，包括关键词扫描、关注流扫描和固定博主。

保存位置通常在情报库下：

```text
<你的 Vault>/xhs/
<你的 Vault>/专辑/
```

## B 站工作流

第一次建议顺序：

1. 打开 `哔哩哔哩工具`。
2. 点击右上角 `配置 Cookie`，优先使用一键获取。
3. 在 `动态追踪` 中跑全关注流或指定 UP 预览。
4. 在 `收藏整理` 中读取收藏夹并选择内容入库。
5. 在 `关注监控` 中配置常驻关键词、智能分组或固定 UP。

## 定时任务与主动工具的关系

ABO 的规则是：定时任务不应该另写一套平行实现。

如果小红书、B 站、论文 Follow Up 等功能已经有主动工具页，那么每日情报或 Feed 监控必须复用主动工具里的：

- 前端组件
- 后端抓取逻辑
- 卡片结构
- 保存路径
- 入库逻辑
- 过滤配置
- 页数、条数、时间窗等参数
- 调试入口

所以遇到问题时，先去主动工具页手动跑一次。如果主动工具能跑通但 Feed 不一致，这通常应视为 bug，并修成主动工具为单一事实来源。

## 常见问题

`今日情报没有内容`
先到小红书、B 站或论文追踪工具页手动跑一次。没有跑过工具或定时任务时，Feed 为空是正常的。

`Cookie 一键配置失败`
确认浏览器已经登录对应平台。小红书和 B 站依赖真实浏览器登录态；登录过期时需要重新获取。

`保存失败`
先检查情报库或文献库路径是否可读写。情报库用于社媒和收藏，文献库用于论文。

`论文 Follow Up 太慢或失败`
Semantic Scholar API Key 可选。大量使用时建议填写自己的 Key；少量使用可以先留空。

`不知道在哪里改监控词`
优先去对应主动工具页。模块管理页只看运行状态、历史记录和诊断，不是精细配置入口。

`想暂时关掉某个来源`
去模块管理暂停模块，或在设置的今日情报偏好里隐藏来源。
