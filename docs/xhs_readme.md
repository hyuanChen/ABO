 完整版设计说明



 一、产品定位

 Another Brain Odyssey 是一个面向长期成长的个人智能系统。

 它把你每天接触到的论文、平台内容、热点主题、兴趣偏好、生活记录和思考过程，统一

 收回到一个持续积累的个人空间里，让信息不再只是被浏览和遗忘，而是被筛选、被理

 解、被连接、被沉淀，并最终变成真正属于你的知识、判断、行动和人格轨迹。



 它不是单一功能的软件，而是一套围绕“信息输入 - 知识沉淀 - 对话生成 - 生活记录 -

 自我建模”展开的长期工作台。

 它的目标不是帮你多收藏一点，而是帮你长期看清三件事：



1. 你每天到底在接收什么信息。
2. 这些信息如何塑造你的知识结构和判断方式。
3. 长期下来，你会成为一个什么样的人。



 二、它要解决的核心问题

 今天的大多数信息都分散在不同地方：论文平台、短视频平台、内容社区、收藏夹、浏览

 记录、日常手记、临时灵感。你会不断看到很多内容，但很难形成连续的积累。



 这会带来四个典型问题：



1. 信息很多，但真正留下来的很少。
2. 收藏很多，但很难回看，更难形成自己的结构。
3. 看过很多内容，却没有稳定地转化成知识、判断和想法。
4. 生活和研究、兴趣和成长、娱乐和精神状态彼此分离，最终无法看见完整的自己。



 ABO 想做的，就是把这些分散的信息与行为重新串起来，形成一个连续的个人系统。



 三、产品的整体工作逻辑

 这款产品的核心不是某一个页面，而是一条完整的闭环：



1. 收集

   你可以主动获取，也可以让系统按日持续获取你关心的内容，包括论文、平台动态、

 收藏内容、关键词结果和热点主题。

2. 汇聚

   所有新进入系统的内容，会先进入统一的今日情报区，形成你当天的信息入口。

3. 筛选

   你可以根据来源、主题、热度、类型和价值，快速判断哪些值得读、哪些值得留下、

 哪些值得继续追踪。

4. 沉淀

   有价值的信息会被写入情报库、文献库和 Wiki，逐步从“今天看到的内容”变成“以后

 还能复用的知识”。

5. 对话与生成

   当内容进入知识系统之后，你就可以基于这些上下文和助手对话，用模板化方式完成

 总结、过滤、复盘、观点提炼和 idea 生成。

6. 回顾与建模

   系统持续记录你的阅读、偏好、互动、生活和状态变化，最终生成你的注意力地图、

 成长轨迹和个人画像。



 换句话说，ABO 不是“信息采集器”，而是“信息如何进入你、如何留在你、如何继续塑造

 你”的完整过程。

## 一、研究与知识系统

### 1.1 一句话定义

把论文和高价值情报，从“发现”推进到“阅读”，再从“阅读”推进到“知识沉淀”和“idea 生成”。

### 1.2 主链条

`论文追踪 / 主动搜索`
-> `今日情报`
-> `筛选值得读的内容`
-> `深入阅读`
-> `写入文献库 / 写入 Wiki`
-> `和助手对话`
-> `生成过滤结果 / 研究判断 / 新 idea`
-> `形成长期研究地图`

### 1.3 分段解释

#### ① 发现输入

这一段负责把你需要的研究内容持续带进系统。

来源包括：

- 你主动检索到的论文
- 系统每日为你更新的论文
- 基于关键词、主题和延伸线索找到的新材料

这一层的目标不是“抓更多”，而是稳定地把和你研究方向相关的输入带进来。

#### ② 汇入今日情报

新发现的论文和研究线索不会直接沉到底层资料库，而是先进入 `今日情报`。

这样做的作用是：

- 先把当天的新输入集中展示
- 让你快速判断哪些值得进一步看
- 避免所有内容一进来就堆积到底层库里，造成噪音

`今日情报` 在这里扮演的是研究输入的第一层过滤器。

#### ③ 筛选与阅读

这一段是用户真正做判断的地方。

你会在这里完成几件事：

- 看标题、摘要和主题，判断是否值得继续
- 识别哪些论文只是路过信息，哪些值得深入阅读
- 选出真正需要进入长期知识系统的内容

这一步非常关键，因为它决定了系统未来沉淀下来的知识质量。

#### ④ 写入文献库与 Wiki

当一篇论文值得留下时，它会进入两层沉淀：

- `文献库`：负责保留具体论文材料与阅读积累
- `Wiki`：负责把论文中的方法、问题、结论和研究主线，组织成可连接的知识结构

也就是说：

- 文献库更像材料层
- Wiki 更像结构层

这一步完成后，内容就不再只是“今天读过”，而是正式变成你知识系统的一部分。

#### ⑤ 基于知识上下文与助手对话

当论文和研究内容已经和 Wiki 中已有知识连接起来之后，助手的价值才真正出现。

因为这时对话不是基于空白输入，而是基于：

- 你今天筛过的内容
- 你已经保留的论文
- 你已有的知识结构
- 你过去关心过的研究主题

在这个基础上，助手可以继续帮你做：

- 信息过滤
- 多篇内容的归纳与对比
- 研究重点判断
- 下一步阅读建议
- idea 的延伸和生成

#### ⑥ 输出结果

这一条链最后的产出，不是“又存了一篇论文”，而是：

- 一组更清晰的研究重点
- 一份更可执行的阅读与推进计划
- 一批进入 Wiki 的稳定知识节点
- 一些真正有价值的新想法

#### ⑦ 长期价值

长期运行之后，这条链形成的不是论文堆，而是一张不断扩展的研究地图：

- 你追踪了哪些方向
- 你对哪些问题持续投入
- 你形成了哪些长期判断
- 你的研究兴趣如何演化

### 1.4 对应页面

- `论文追踪`
- `今日情报`
- `文献库`
- `Wiki`
- `助手`

### 1.5 这条链的本质

`把研究输入 -> 变成知识结构 -> 再变成判断与 idea`

## 二、注意力与生活输入系统

### 2.1 一句话定义

把你在小红书和哔哩哔哩里的主动与被动信息，整理成每天的注意力记录、生活日报和长期偏好画像。

### 2.2 主链条

`平台输入（小红书 / 哔哩哔哩）`
-> `主动信息 + 被动信息一起进入系统`
-> `按类别 / 主题 / 热点整理`
-> `记录当天停留与偏好`
-> `筛出值得保留的信息`
-> `生成每日日报`
-> `沉淀长期注意力画像`
-> `看见自己正在被什么塑造`

### 2.3 分段解释

#### ① 平台输入

这一条链的输入，主要来自两个平台：

- `小红书`
- `哔哩哔哩`

而且它不是只记录你主动点进去的内容，还包括平台持续推到你面前的内容。

所以它记录的是两种输入：

- `主动信息`：你主动收藏、主动关注、主动点开的内容
- `被动信息`：平台推荐给你、但你仍然停留或关注的内容

这两种信息一起，才能更真实地反映你的注意力。

#### ② 同类主题与热点整理

平台内容被带进来之后，不应该只是原样堆叠。

这一层要做的是：

- 按内容类别归类
- 按主题聚合
- 按热点趋势整理
- 识别哪些主题在同一时期反复出现

这样一来，系统记录的就不只是“今天看到了什么”，而是“今天你的注意力集中在哪里”。

#### ③ 记录每日注意力偏好

这一段是第二条链真正的核心。

它记录的不是单条内容本身，而是：

- 你今天在哪些主题上停留更多
- 你反复回看的是什么
- 你愿意留下来的是什么
- 你每天真正吸收进去的内容是什么

这一步的意义在于，让注意力不再消失在平台流里，而成为一个可回看的长期记录。

#### ④ 筛出值得保留的情报

在平台内容里，并不是所有东西都需要进入长期系统。

真正要被保留下来的，通常是：

- 有代表性的主题
- 有持续价值的内容
- 能反映你偏好的线索
- 会在之后继续影响你判断和生活方向的内容

这些内容会进入你的 `情报库`，作为长期的个人情报资产。

#### ⑤ 生成每日日报

第二条链不只关心信息，也关心生活。

系统会把你当天的娱乐内容、精神触动、审美偏好和平台输入，一起整理成日报。

这里的日报不是普通任务汇报，而是更接近：

- 今天我接收了什么
- 今天什么内容打动了我
- 今天我的注意力偏向哪里
- 今天我在精神上被什么滋养

这才是“真正和生活有关”的部分。

#### ⑥ 沉淀长期注意力画像

当这些每日记录不断累积之后，系统就能逐渐呈现：

- 你长期关注什么
- 你容易被什么内容吸引
- 你在什么主题上反复投入时间
- 你的审美、兴趣、精神取向如何变化

这时候产品记录的，就不只是信息历史，而是你的注意力人格。

### 2.4 对应页面

- `小红书工具`
- `哔哩哔哩工具`
- `今日情报`
- `情报库`
- `手记`
- `数据洞察`

### 2.5 这条链的本质

`把平台输入 -> 变成注意力记录 -> 再变成生活日报与长期偏好画像`

## 三、个人管理与虚拟陪伴系统

### 3.1 一句话定义

把知识资产、生活状态、习惯规律和长期记录统一收回到角色主页，形成一个会总结、会建议、会陪伴的个人系统。

### 3.2 主链条

`文献库 / 情报库 / 手记 / 作息 / 运动 / 日常状态`
-> `统一汇总到角色主页`
-> `形成每日状态与阶段记录`
-> `识别规律与变化`
-> `给出建议与总结`
-> `生成每周 / 长期回顾`
-> `塑造虚拟角色陪伴`
-> `完成 another brain odyssey 的长期旅程`

### 3.3 分段解释

#### ① 收拢个人材料

这一条链首先做的，是把原本分散的个人信息收拢回来。

包括：

- 文献库中的研究积累
- 情报库中的日常信息资产
- 手记中的生活记录
- 作息、习惯、运动等生活状态
- 每天留下来的行为痕迹

这些东西原本可能分散在很多地方，但在这里会被统一看待。

#### ② 汇总到角色主页

角色主页是第三条链的中心。

它不是一个静态个人档案，而是一个动态的“你目前的状态面板”。

这里会持续整理出：

- 你是谁
- 你最近在做什么
- 你最近状态怎么样
- 你接下来最该推进什么

也就是说，它负责把所有输入转成“当前的自己”的可见形态。

#### ③ 形成每日状态记录

当日常记录不断进入系统之后，角色主页就可以呈现出每日层面的变化：

- 今天的状态
- 今天的重点
- 今天的任务
- 今天的轨迹
- 今天和昨天相比的变化

这样你不再只是在做事，而是在被持续记录。

#### ④ 识别规律与变化

当记录跨越更长时间之后，系统会开始出现第二层价值：

- 识别你的生活规律
- 识别你的工作节奏
- 识别你的输入和状态之间的关系
- 识别你的成长趋势和波动

这时产品就不仅在“记”，也开始在“理解”。

#### ⑤ 给出建议与总结

第三条链不只是展示记录，还应该给出反馈。

比如：

- 对每日生活给出建议
- 对每周状态给出总结
- 对当前节奏给出提醒
- 对下一步推进方向给出提示

这一步决定了它是不是一个真正能陪用户长期走下去的系统。

#### ⑥ 形成虚拟角色陪伴

当系统持续记录你的输入、偏好、状态、节奏和成长轨迹之后，它就会越来越像一个“有连续记忆的陪伴者”。

它知道：

- 你关注过什么
- 你坚持过什么
- 你容易被什么吸引
- 你过去处于什么状态
- 你现在和过去相比发生了什么变化

这就是“虚拟角色陪伴”的真正基础。

它不是一个装饰性角色，而是一个基于长期记录形成的、越来越理解你的镜像系统。

#### ⑦ 长期价值

长期来看，这一条链会让用户获得三层东西：

- 一个完整的个人管理空间
- 一套不会遗忘的生活和成长记录
- 一个越来越像“另一个自己”的虚拟陪伴角色

### 3.4 对应页面

- `角色主页`
- `助手`
- `手记`
- `健康管理`
- `数据洞察`
- `情报库`
- `文献库`

### 3.5 这条链的本质

`把个人记录 -> 变成状态管理 -> 再变成长期陪伴与自我建模`

## 六、给核心功能 HTML 直接使用的整理版

这一节不是继续展开页面介绍，而是把三大核心功能重新整理成更适合直接画成逻辑图的底稿。

统一要求只有六件事：

1. 先说清楚输入从哪里来
2. 再说清楚先汇到哪里
3. 再说清楚关键判断发生在哪里
4. 再说清楚最终沉淀到哪里
5. 再说清楚会输出什么结果
6. 最后说清楚长期价值是什么

这样无论后面是继续写官网、做 PPT，还是拆成单独的 HTML canvas 图，逻辑都会更稳。

### 6.1 ABO 的统一递进关系

ABO 不是三个并列页面，也不是三个彼此分离的工具，而是一套递进系统：

- 第一条链回答的是：`我在接收什么高价值知识`
- 第二条链回答的是：`我在被什么内容持续吸引`
- 第三条链回答的是：`这些输入最终把我塑造成什么样的人`

所以它们之间真正的关系是：

`我接收什么`
-> `我留下什么`
-> `我理解什么`
-> `我如何行动`
-> `我最终成为谁`

### 6.2 三条主链的标准表达

#### A. 研究与知识系统

一句话定义：

`把论文发现、推送、入库、digest 编译、Wiki 沉淀和助手写作串成一条研究工作流，让论文不只是被保存，而是被持续转成你的研究体系。`

标准链条：

`arXiv 关键词追踪 / Semantic Scholar follow up`
-> `系统推送到研究入口`
-> `入库到 raw/ 与文献库`
-> `保留摘要与 intro 作为 digest 素材`
-> `生成日报 / AI 打分 / 技能标签`
-> `编译为 Wiki / 技能地图`
-> `Obsidian 插件兼容与额外可视化`
-> `助手写作与研究推进`
-> `输出写回 wiki/`
-> `↻ 下一轮迭代`

分段说明：

- `输入来源`：`arXiv` 关键词追踪、`Semantic Scholar` follow up、引用链延伸、作者跟进和系统推送
- `先汇到哪里`：先进入研究入口层，再决定哪些内容应该进入入库与跟进链，而不是所有论文都直接沉到知识层
- `入库策略`：论文进入 `raw/` 与 `文献库`，同时稳定保留摘要与 `Introduction`，作为后续 agent digest、日报生成和写作的原材料
- `中间编译`：在入库之后生成日报、AI 打分、优先级排序和技能标签，帮助系统判断什么该先读、什么该先写、什么该先跟进
- `沉淀位置`：论文卡片、概念页、related work 连接、研究空白和技能地图都会继续写入 `Wiki`
- `可视化层`：兼容 `Obsidian` 插件生态，支持图谱、Canvas、Dataview 和额外研究可视化
- `助手作用`：助手基于 digest、日报、技能地图和 Wiki 结构继续做摘要写作、综述整理、研究判断和下一步行动规划
- `反馈闭环`：AI 打分、日报结果和 Wiki 更新会反向影响下一轮推送、follow up 优先级和阅读顺序
- `直接输出`：paper digest、研究日报、技能地图、写作草稿、研究判断和下一轮任务清单
- `长期价值`：最后形成的不是论文堆，而是一套会持续编译、持续可视化、持续辅助写作的研究系统

对应页面：

- `论文追踪`
- `今日情报`
- `文献库`
- `Wiki`
- `助手`

#### B. 注意力与生活输入系统

一句话定义：

`把平台上的主动输入与同类分组统一推送的被动输入，一起变成注意力记录、情报资产、个人 Wiki 和长期偏好画像。`

标准链条：

`小红书 / 哔哩哔哩平台输入`
-> `主动输入 + 同类分组统一推送的被动系统`
-> `按类别 / 主题 / 热点整理`
-> `记录当天停留与偏好`
-> `筛出值得保留的情报`
-> `写入情报库`
-> `编入个人 Wiki`
-> `生成每日日报与生活记录`
-> `反馈到下一轮推送、筛选与助手理解`
-> `沉淀长期注意力画像`

分段说明：

- `输入来源`：收藏、关注、动态、推荐流、热点主题、你主动点开与平台持续分组推送给你的同类内容
- `先汇到哪里`：先进入系统里的当日信息入口，再按主题、类别和热点做整理，尤其把被动输入收敛成可判断的主题包
- `核心判断`：判断哪些内容只是短暂停留，哪些真正代表你的偏好、长期关注与精神走向
- `沉淀位置`：高价值内容先进入 `情报库`，再继续编入 `个人 Wiki`；日常精神输入和感受进入 `手记` 与日报
- `直接输出`：当天注意力分布、精神触动、热点聚类、生活日报，以及逐渐稳定的个人主题网络
- `反馈闭环`：情报库和 Wiki 会反过来影响下一轮筛选、分组推送理解和助手整理能力，让系统越用越像你
- `长期价值`：看见自己反复被什么吸引，并把这些输入沉淀为真正属于你的个人体系，而不只是平台痕迹

对应页面：

- `小红书工具`
- `哔哩哔哩工具`
- `今日情报`
- `情报库`
- `Wiki`
- `手记`
- `数据洞察`

#### C. 个人管理与虚拟陪伴系统

一句话定义：

`把知识资产、生活状态、习惯规律和长期记录统一收回到角色主页，形成一个会总结、会建议、会陪伴的个人系统。`

标准链条：

`文献库 / 情报库 / 手记 / 作息 / 运动 / 日常状态`
-> `统一汇总到角色主页`
-> `形成每日状态与阶段记录`
-> `识别规律与变化`
-> `给出建议与总结`
-> `生成每周 / 长期回顾`
-> `塑造虚拟角色陪伴`

分段说明：

- `输入来源`：研究积累、日常情报、手记、健康管理、作息习惯、每日行为痕迹
- `先汇到哪里`：统一被收拢到 `角色主页`，变成当前自我的状态面板
- `核心判断`：识别最近状态、推进重点、生活规律、波动来源与成长趋势
- `沉淀位置`：每日状态记录、阶段总结、回顾文档、个人画像
- `直接输出`：建议、提醒、每周总结、长期回顾、阶段变化说明
- `长期价值`：形成一个越来越理解你的虚拟角色与镜像系统，陪你走完整段 Another Brain Odyssey

对应页面：

- `角色主页`
- `助手`
- `手记`
- `健康管理`
- `数据洞察`
- `情报库`
- `文献库`

### 6.3 画成 HTML 逻辑图时的统一写法

为了让三张图风格统一，后面拆 HTML 时建议每张图都保持同一种表达顺序：

每一张图都尽量做到：

- 一眼能看懂输入从哪里来

## 七、ABO 代言人角色设定与 GPT Image 提示词

这一节是给 ABO 设计一个可以长期使用的 2D 代言角色。

这个角色不是普通的“吉祥物”，而应该像一个真正陪你长期使用产品的研究陪跑者：

- 她要能代表 `研究与知识系统`
- 她要能代表 `注意力与生活输入系统`
- 她要能代表 `个人管理与虚拟陪伴系统`

同时，她还要和 ABO 现在这套视觉气质统一：

- 温暖
- 轻柔
- 可控
- 清醒
- 可沉淀
- 有陪伴感
- 有本地知识库和个人研究工作台的感觉
- 不像营销 IP，也不像夸张二次元偶像

### 7.1 角色一句话定位

`一个会帮你记住、整理、理解并温柔提醒你的 2D 女性研究陪跑员。`

### 7.2 建议昵称

可以先用：

- `阿波`
- `Abo`
- `ABO 陪跑员`

其中 `阿波` 最适合日常称呼，天然有一点亲近感，也不会太硬。

### 7.3 角色气质关键词

- `温暖但清醒`
- `聪明但不冷`
- `可爱但不幼稚`
- `贴心但不过度黏人`
- `像会长期陪你工作的研究搭子`
- `像另一个大脑的人格化形象`
- `像帮你把生活和知识都轻轻收好的那个人`

### 7.4 角色外观设计方向

#### 整体年龄与比例

- 年轻女性角色
- 视觉年龄大约 `20-24 岁`
- 明显可爱，但不是儿童感
- 比例偏可爱，建议 `5 到 5.5 头身`
- 不能做成夸张大头 Q 版，也不要太成熟御姐
- 要像“年轻、聪明、亲和、会陪你长期工作的人”

#### 脸部感觉

- 脸型偏圆润，轮廓柔和
- 眼睛较大，但不是夸张动漫大眼
- 眼神要有“观察、理解、耐心倾听”的感觉
- 表情基调是温柔、安静、聪明、轻轻带笑
- 不要做成卖萌过度、wink 过度、偶像营业感
- 更像是“知道你今天状态怎么样，也知道你下一步该做什么”的感觉

#### 发型

- 中短发或锁骨发最合适
- 头发蓬松柔软，发尾略微内扣或轻卷
- 主色建议 `柔和栗棕色 / 奶茶棕 / 浅胡桃棕`
- 可以在发丝高光里带一点很淡的 `雾感紫灰` 或 `暖米金`
- 刘海不要太厚重，留一些轻盈空气感
- 可以带一个非常小的侧边编发，增加整理感和温柔感
- 发饰建议使用 `圆角书签夹 / 圆角便签夹 / 小卡片 tab` 这种和知识整理有关的造型

#### 服装基调

她不应该像魔法少女、战斗角色、女仆、偶像，也不应该像硬邦邦的实验室白大褂。

她更适合：

- `轻柔研究工作台风格`
- `圆角卡片感服装结构`
- `带一点日常开衫 / 短斗篷 / 轻便外套 / 层叠裙摆`
- `像把知识卡片、日记页和个人工作台穿在身上的感觉`

推荐服装组合：

- 上半身是一件 `奶油白 / 温暖米白` 的短外套或开衫
- 袖子略微宽松，边缘圆润
- 领口可以带一点 `薄荷绿` 或 `浅灰蓝` 的内搭层次
- 胸前可以有一个 `三色小徽章`，象征三个功能系统
- 下半身用 `浅薰衣草灰` 或 `雾粉紫` 的裙装，线条简洁，不要复杂花边
- 腰间或侧腰可以挂一个 `圆角小笔记本 / 卡片夹 / 小档案包`
- 鞋子可以是 `浅棕 / 奶茶色` 的小皮鞋或短靴，整体轻巧

#### 配色方案

角色必须和 ABO 现有页面里的色彩逻辑一致，不能做成高饱和花哨角色。

核心配色建议：

- `主底色`：温暖米白、纸张白、奶油白
- `辅助色 1`：薄荷绿、柔和鼠尾草绿
- `辅助色 2`：浅灰蓝、雾霾蓝
- `辅助色 3`：雾感薰衣草紫
- `辅助色 4`：低饱和豆沙粉 / 尘粉色
- `点缀色`：少量蜂蜜琥珀色 / 金棕色

这些颜色分别可以映射 ABO 的三条链：

- `研究与知识系统`：薄荷绿 + 蓝色 + 紫色
- `注意力与生活输入系统`：豆沙粉 + 柔蓝 + 绿色
- `个人管理与虚拟陪伴系统`：蓝色 + 紫色 + 琥珀色 + 绿色

#### 圆角与 UI 语义如何进入角色

这个角色最重要的一点，是要让人看出她属于 ABO 这套圆角、轻柔、卡片式的 UI 世界。

所以她的设计里要有这些“圆角视觉语言”：

- 发夹像 `圆角书签 tab`
- 徽章像 `圆角状态标签`
- 小包像 `圆角笔记卡盒`
- 裙摆或外套分层像 `叠起来的圆角卡片`
- 袖口、口袋、包边都尽量偏圆润
- 身边漂浮的小道具也应是 `圆角卡片 / 便签 / 小面板`

这样角色会自然像从产品里长出来，而不是外接的 IP。

### 7.5 三个功能如何映射到角色身上

#### A. 研究与知识系统

这一部分要让人感觉她会“发现、整理、沉淀知识”。

可以通过这些元素体现：

- 手里拿着一个 `圆角笔记板 / 小型研究手帐 / 透明圆角平板`
- 身边漂浮几张 `小论文卡片 / 摘要卡 / 便签卡`
- 有一个小小的 `书签夹 / 放大镜意象 / 研究标签`
- 气质上像是“先帮你发现重点，再帮你整理进知识系统”

#### B. 注意力与生活输入系统

这一部分要让人感觉她会“轻柔地接住你每天看到的东西”，而不是只会做学术。

可以通过这些元素体现：

- 一枚很小的 `日记页吊饰 / 生活记录 charm`
- 一些柔和漂浮的小图标，例如 `便签、日历页、气泡、叶片、心形但不要俗气`
- 神态上更像“她会记得你今天被什么打动”
- 她不是只懂论文，她也懂生活、兴趣、审美和精神输入

#### C. 个人管理与虚拟陪伴系统

这一部分要让人感觉她不仅记录你，还会理解你、提醒你、陪你往前走。

可以通过这些元素体现：

- 胸前或腰间有 `三枚小圆角徽章`，象征状态、回顾、陪伴
- 身边可以有一个很轻的 `轨道光环 / 小星点 / 状态环`
- 姿态略微前倾，像在认真听你说话
- 一只手可以是 `轻轻向前展开`，有接住感
- 她整体要有“不会打扰你，但会一直在”的陪伴氛围

### 7.6 角色应该给人的第一印象

第一眼看到她，应该立刻让人联想到：

- 这是一个和知识整理有关的角色
- 这是一个很温暖、很会倾听的角色
- 这是一个会帮我把混乱输入慢慢收拢起来的角色
- 她不只是可爱，她是真的“有用”
- 她像 ABO 的人格化版本

### 7.7 不应该出现的方向

下面这些方向尽量避免：

- 太像偶像企划角色
- 太像游戏战斗角色
- 太像魔法少女
- 太像儿童绘本低龄角色
- 太科技、太赛博、太荧光 neon
- 太性感、太成熟、太御姐
- 太商务、太僵硬、太企业吉祥物
- 太复杂，身上挂满太多装饰
- 太像“AI 助手模板角色”

她应该是：

- `原创`
- `克制`
- `柔和`
- `有记忆感`
- `有知识感`
- `有陪伴感`

### 7.9 如果第一张图出来后要继续微调，可以追加的短句

如果第一次生成已经接近了，但还想更贴合 ABO，可以在后面继续补这些方向：

- `让角色更像长期研究搭子，而不是普通可爱二次元角色`
- `让圆角卡片和知识整理感再明确一点，但不要增加复杂道具`
- `让温暖和贴心感更强一点，像会认真听你说话的人`
- `减少偶像感，增加陪跑感、清醒感和本地知识库气质`
- `让角色更像 ABO 的人格化版本，而不是通用 AI 助手`

### 7.10 最终目标

如果这个角色设计成功，她应该达到下面这个效果：

`她一出现，用户就会觉得：这个产品不是冷的，它会记得我，会帮我整理，会慢慢理解我，也会一直陪我往前走。`

### 7.11 后续资产生成顺序建议

为了让角色一致性更高，建议生成顺序是：

1. 先生成 `主视觉全身角色`
2. 再用主视觉图去生成 `三视图版`
3. 再用主视觉图去生成 `表情包版`
4. 最后再生成 `应用内引导版`

这样后面出来的角色稳定性会更高，不容易每一张都长得不一样。

### 7.12 后续所有 prompt 通用的角色一致性锁定词

后面无论你生成表情包、三视图，还是应用内引导图，都建议先在 prompt 最前面加这一段：

```text
Input images: Image 1 = previously generated ABO mascot reference image, use it as the primary identity reference and preserve the exact same character identity.

Identity lock: keep the exact same face shape, same eye shape, same warm gentle expression baseline, same hair length and hairstyle, same hair color, same rounded UI-inspired outfit, same color palette, same rounded accessories, same soft paper-like illustration feeling, same lovable and reliable personality. This must be the exact same ABO mascot character, not a similar new character.
```

如果后面你发现角色还是会跑偏，还可以继续补一句：

```text
Do not redesign the character. Preserve identity first, then change only pose / expression / scene.
```

### 7.13 表情包版 prompt

这一版的目标，不是做普通卖萌贴纸，而是做“ABO 角色的实用陪伴表情包”。

它应该体现：

- 可爱
- 贴心
- 有点轻松
- 但不幼稚
- 能用于产品宣传、社媒、应用内反馈
- 每个表情都像她真的在陪你工作

建议做成 `8 格或 9 格角色表情贴纸页`。

#### 表情主题建议

可以优先做这 8 个：

1. `我来帮你整理`
2. `今天先看这个`
3. `有新论文来了`
4. `先记下来`
5. `你已经做得很好了`
6. `稍微休息一下`
7. `今晚记得复盘`
8. `我陪你继续`

### 7.14 三视图版 prompt

这一版的目标，是固定角色设定，方便后续继续出更多图。

三视图不需要很花哨，它最重要的是：

- 同一个角色的正面、侧面、背面都清楚
- 比例、服装结构、配色关系都稳定
- 道具和徽章位置能看明白
- 后续任何插图都可以拿它当官方设定参考

#### 可直接给 GPT Image 的 prompt

```text
Use case: stylized-concept
Asset type: character turnaround sheet / model sheet for ABO mascot
Input images: Image 1 = previously generated ABO mascot reference image, preserve the exact same character identity
Primary request: 为已经生成好的 ABO 女性代言角色制作一张完整的三视图角色设定图。必须是同一个角色，不能重新设计。需要清楚展示角色的正面、侧面、背面三种视角，并尽量保持同一比例、同一服装结构、同一发型和同一配色。

Scene/backdrop: 纯净浅米白或浅纸白背景，类似专业角色设定图背景。允许极淡的纸面肌理，但不允许复杂场景。不要多余装饰，不要漂浮卡片，不要剧情化背景。核心是角色结构清晰。

Subject: 同一个 ABO 女性角色，20-24 岁视觉年龄，5 到 5.5 头身可爱比例，圆润柔和、温暖、聪明、可靠。请完整保留主视觉中的脸型、眼睛、发型、发色、服装、配色、腰间配件、圆角徽章、圆角书签发饰等核心识别特征。

Composition/framing: 一张角色设定图中并排展示 3 个完整全身角色：
1. front view 正面
2. side view 侧面
3. back view 背面
三个人物站姿统一，中性自然站立，不夸张摆 pose。角色大小一致，脚底基线一致，方便后续作为标准设定参考。可以在画面角落额外加一个很小的头像特写或配件特写，但如果会影响稳定性，也可以不加。不要文字标签，不要 logo，不要水印。

Pose and body language: 标准设定站姿，双臂自然下垂或轻微放松，不做表演动作。重点是结构清楚。人物不能被裁切，衣服层次、裙摆、外套边缘、包袋、徽章、发饰都要看得清。

Design language: 继续强调 ABO 的圆角 UI 语言。服装分层像圆角卡片叠层，徽章像圆角状态标签，腰包像圆角卡片盒，整体边缘柔和，没有尖锐硬朗装甲感。要让人一眼看出来这是一个和知识整理、陪伴、轻柔工作台有关的角色。

Style/medium: 高质量 2D 角色设定图，清晰线稿，专业设定稿感觉，但仍然保留温暖柔和的品牌插画气质。不要写实，不要 3D，不要游戏建模图。要像品牌官方发布的角色设定 sheet。

Color palette: 完全沿用主视觉角色的配色，温暖米白、薄荷绿、浅灰蓝、雾紫、豆沙粉、少量蜂蜜琥珀点缀。颜色要稳定统一，不能因为换视角就改变服装色彩关系。

Important goal: 这张图的重点不是“漂亮动作”，而是“官方设定稳定”。看到这张图时，应该让后续任何画师或模型都明白：这就是 ABO 代言人的标准角色设定。

Constraints: exact same character identity, exact same outfit, exact same hairstyle, exact same accessories, exact same palette, no redesign, no dramatic pose, no complex background, no extra props, no text, no logo, no watermark

Avoid: fashion pose, dynamic action pose, redesign, alternate costume, alternate hairstyle, exaggerated body proportions, elaborate background, 3D look, model turnaround with text labels, technical blueprint clutter
```

### 7.15 应用内引导版 prompt

这一版的目标，是把角色真正放进 ABO 产品体验里。

不是做一张普通立绘，而是做“角色 + 引导场景”的组合图。

最适合直接做成 `3 张 onboarding 插图`，分别对应三大功能：

1. `研究与知识系统引导页`
2. `注意力与生活输入系统引导页`
3. `个人管理与虚拟陪伴系统引导页`

这样角色就不只是一个代言人，而是一个真的在产品里陪用户理解功能的人。

### 7.16 这三组后续 prompt 的用途

如果主视觉角色已经满意，那么这三组 prompt 基本可以覆盖你后面最常见的角色资产需求：

- `表情包版`：用于社媒传播、应用内反馈、小红书封面配角、产品气泡反馈
- `三视图版`：用于固定角色设定，后面继续出图不跑偏
- `应用内引导版`：用于 onboarding、产品介绍页、官网角色说明图

### 7.17 一句话执行建议

最稳的顺序就是：

`先主视觉 -> 再三视图锁设定 -> 再表情包扩展 -> 再做应用内场景图`

## 八、ABO 无性别 Cartoon 角色重设版

这一版是对上面角色设定的重新整理版。

如果你后面决定把 ABO 的代言形象收成 `cartoon 角色`，并且 `不设定性别`，那么后续生成请优先使用这一节。

也就是说：

- 第七节可以保留作为旧方案参考
- 第八节是新的推荐方案
- 如果两者有冲突，以 `第八节` 为准

### 8.1 新版本角色一句话定位

`一个无性别、温暖、圆角、会陪你整理知识与生活输入的 cartoon 陪跑精灵。`

### 8.2 角色核心变化

这一版和上面最大的区别是：

1. 不再设定为女性角色
2. 改为 `cartoon mascot / cartoon companion` 方向
3. 更像从 ABO 的圆角 UI 和知识卡片世界里长出来的角色
4. 更适合长期做品牌代言、表情包、onboarding 插图和产品内角色引导

### 8.3 新角色的整体方向

这个角色不应该是“二次元少女”或“拟人助手”。

它更适合是一个：

- `无性别`
- `温暖`
- `聪明`
- `安静`
- `很会收纳和整理`
- `像小小知识精灵`
- `像个人工作台里一直陪着你的角色`
- `像另一个大脑的人格化 cartoon 版本`

它应该让人感觉：

- 可爱
- 贴心
- 不黏人
- 不夸张
- 不商业 mascot 化
- 不低龄幼稚
- 但一眼就会让人有亲近感

### 8.4 外形设计建议

#### 整体体型

- 比例偏 cartoon，建议 `3.8 到 4.5 头身`
- 身体结构简单、圆润、轻巧
- 不要真实人体比例
- 不要明显男性或女性身体特征
- 没有强调胸、腰、胯等任何性别化轮廓
- 更像 `圆角小精灵 + 卡片工作台助手`

#### 头部与脸

- 头部偏大，但不能过度婴儿化
- 头部轮廓圆润，像一张圆角卡片或柔软小便签的变体
- 眼睛偏大，但不要夸张动漫眼
- 眼神要温和、理解、聪明、安静
- 嘴巴很小，表情细腻，不做夸张颜艺
- 脸部不强调性别化五官
- 整体像一个“懂你、记得你、会轻轻提醒你”的角色

#### 头顶特征

这一版不建议保留明显的人类发型。

更推荐做成一种 `标志性的小头顶结构`：

- 像 `书签 tab`
- 像 `圆角便签折角`
- 像 `翻开的卡片页角`
- 或像一小撮软软的纸页冠饰

这个特征会非常重要，因为它会成为 ABO 角色最稳定的识别点。

#### 身体和服装

这个角色最好不是穿完整的人类衣服，而是穿一种介于 `小斗篷 / 小外套 / 小工作围层 / 卡片层叠壳` 之间的结构。

可以理解成：

- 身体像一个圆润的小工作精灵
- 外层包着一件圆角短斗篷或圆角工作披层
- 衣摆和边缘像叠起来的卡片
- 前胸有 3 枚小圆角徽章，对应 3 个核心功能
- 侧边挂一个小小的 `圆角卡片包 / 小笔记盒 / 小档案袋`

整体不要做成真实衣服褶皱很多的角色。

更像：

- `UI 卡片人格化`
- `知识卡片精灵`
- `研究工作台助手`

#### 手和道具

- 手可以短短软软，但依然有表达动作能力
- 适合拿圆角卡片、圆角笔记板、小手帐、小面板
- 不要拿魔法杖
- 不要拿武器
- 不要拿夸张工具
- 道具应该始终服务于“整理、记录、陪伴、提示”

### 8.5 色系要求

虽然角色不设定性别，但色系最好还是和 ABO 原有气质保持关联。

不一定要原封不动照搬原来的浅色系，更推荐：

- 以 ABO 原有色系为气质基础
- 但整体 `更深一点`
- 更有存在感一点
- 在白底或浅背景里更容易被一眼看见

建议继续参考这些颜色方向：

- `主底色`：温暖米白、纸张白、奶油白
- `研究感`：薄荷绿、柔和鼠尾草绿
- `整理感`：浅灰蓝、雾霾蓝
- `知识沉淀感`：雾感薰衣草紫
- `生活输入感`：低饱和豆沙粉
- `提醒与重点`：少量蜂蜜琥珀色 / 金棕色

整体原则：

- 比原来更深一点
- 比原来更显眼一点
- 温暖
- 纸面感
- 轻柔
- 不荧光
- 不霓虹
- 可以有更清楚的主辅色对比

### 8.6 UI 圆角语言怎么进入角色

这一版 cartoon 角色最关键的，不是“长得像谁”，而是要让人一眼看出它来自 ABO 的 UI 世界。

所以角色身上一定要有这些圆角语言：

- 头顶像 `圆角页签`
- 身体外层像 `圆角卡片叠层`
- 徽章像 `圆角标签`
- 小包像 `圆角卡盒`
- 漂浮道具像 `圆角便签 / 圆角 digest 卡 / 圆角知识节点`
- 所有边缘都偏软、偏圆、偏轻

### 8.7 三个功能如何映射到这个 cartoon 角色

#### A. 研究与知识系统

要让它看起来像：

- 会发现重点
- 会整理摘要
- 会把论文变成 Wiki

对应元素可以是：

- 手里拿着 `圆角论文摘要卡`
- 身边漂浮 `digest 卡片`
- 小小的 `研究标签 / 书签意象`
- 旁边有少量 `技能地图节点`

#### B. 注意力与生活输入系统

要让它看起来像：

- 会接住你每天刷到的东西
- 会帮你把生活输入慢慢收拢

对应元素可以是：

- 小日记页
- 小情报卡
- 小生活记录卡
- 很轻的气泡、叶片、日历页意象

#### C. 个人管理与虚拟陪伴系统

要让它看起来像：

- 会理解你
- 会提醒你
- 会长期陪着你

对应元素可以是：

- 3 枚小圆角徽章
- 很轻的状态环 / 小小轨道圈
- 向前展开的小手势
- 有“接住你”的姿态

### 8.8 角色应该给人的第一感觉

第一眼看到它，应该会觉得：

- 这是 ABO 的角色
- 它和知识整理有关
- 它很温暖
- 它不吵闹
- 它会陪我一起做事情
- 它不是装饰，它是真的“能帮忙”的角色

### 8.9 需要避免的方向

- 太像普通二次元人形角色
- 太像明确男性或女性角色
- 太像儿童启蒙卡通
- 太像企业 mascot 玩偶服
- 太像赛博 AI 助手
- 太像游戏宠物
- 太像魔法小精灵
- 太花哨
- 太荧光
- 太多装饰
- 太多夸张表情

### 8.10 可直接用于 GPT Image 的主 prompt

```text
Use case: stylized-concept
Asset type: brand mascot / spokesperson cartoon character for ABO
Primary request: 设计一个原创的 2D cartoon 精灵角色，用于 ABO（Another Brain Odyssey）的品牌代言形象。这个角色必须是无性别设定，不要明确男性或女性特征。设计目标不是普通吉祥物，而是达到“高质量怪物收集游戏精灵设计”的观感：轮廓强、记忆点强、可爱、耐看、让人想一直带在身边，但不能直接像任何现成 IP 角色。它应该像一个会陪你整理知识、接住生活输入、并长期守护你的陪伴精灵。

Scene/backdrop: 干净简洁的浅暖米白背景，像轻柔纸张和本地知识库工作台的结合。背景里只保留非常淡、非常克制的圆角 UI 痕迹，例如极浅的卡片轮廓、很淡的网格、少量知识节点线索。背景不能复杂，也不要纯白空背景。

Subject: 单个原创 cartoon spirit creature，无性别设定，2D 高质量插画风，整体比例偏 creature mascot，约 3.8 到 4.5 头身，圆润、柔软、毛茸茸、非常有亲近感。不要做人形 mascot，更像一个小型陪伴精灵。它可以坐着、轻轻漂浮、趴着、蜷着或半蹲，不需要站立。整体设计要像“圆角书签 + 小型森林精灵 + 文具知识守护兽”的结合体。轮廓必须一眼就能记住，缩小看也有识别度。

Creature appeal: 参考高质量怪物收集游戏里那种优秀精灵设计语言，但不要直接模仿任何现成角色。角色应该有一个非常清晰的主轮廓、一个稳定的视觉记忆点、可爱的脸、短短软软的四肢或爪爪、柔软蓬松的身体、让人想摸一摸的绒感。它要像“会陪你学习和整理世界的小守护精灵”，而不是普通卡通玩偶。

Color palette: 色系不需要机械地照搬 ABO 原有浅色版本，但要延续同样的温暖、纸面感和知识工作台气质。整体颜色请比原来的页面色系更深一点、更稳一点、更有主次。可以使用更深的鼠尾草绿、灰蓝、雾紫、暖豆沙、蜂蜜琥珀和温暖米白，让角色在浅暖背景上更显眼，但仍然柔和，不要 neon，不要荧光，不要脏重。整体最好有 1 个主色块、1 个辅助色块、1 个小面积高亮点缀色，像优秀精灵设计那样清楚。

Design language: 保留 ABO 的圆角语言，但把这些语言融进精灵本体，而不是做成人类穿搭。身体边缘、耳朵 / 角 / 头顶冠饰 / 尾巴末端 / 胸前花纹，都可以带圆角卡片、书签 tab、便签页角、知识节点的感觉。整体要干净、统一、少而精，不要复杂配件堆砌。

Function symbolism: 只用少量元素暗示三个功能。研究与知识系统可以通过摘要卡、知识纹样、书签感部件来体现；注意力与生活输入系统可以通过小日记卡、小气泡、小记录纹样来体现；个人管理与虚拟陪伴系统可以通过状态徽章、小守护环、柔和陪伴姿态来体现。重点是这些功能像天生长在这个精灵身上，而不是挂一堆道具。

Composition/framing: 单角色为主，完整可见，画面留白充足，方便后续做品牌使用。构图保持简单，不要复杂。可以让角色处在更自然的精灵姿态，比如坐着、趴着、轻轻漂浮、歪头看你。

Important emotional goal: 这个角色最重要的不是复杂设定，而是第一眼就让人觉得“好喜欢、好安心、想带着它”。它应该像一个真正会陪你长期使用 ABO 的精灵伙伴，能让人自然联想到收藏感、陪伴感、守护感和长期记忆感。
```
